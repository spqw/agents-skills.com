01

Install & Add Packages

Install pnpm itself

Three ways to get pnpm on your system.

# Option 1: Corepack (recommended, ships with Node 16.13+)
corepack enable pnpm

# Option 2: Via npm
npm install -g pnpm

# Option 3: Standalone script
curl -fsSL https://get.pnpm.io/install.sh | sh -

# Pin version in your project
corepack use pnpm@latest

Add dependencies

Production, dev, optional, exact, and peer dependencies.

# Install all deps from package.json
pnpm install

# Production dependency
pnpm add react react-dom

# Dev dependency
pnpm add -D typescript @types/react vitest

# Exact version (no ^ or ~)
pnpm add -E lodash@4.17.21

# Optional dependency
pnpm add -O fsevents

# Peer dependency
pnpm add --save-peer react

# From a git repo
pnpm add github:user/repo

# From a local path
pnpm add ./packages/shared
02

Remove & Update

Remove packages

# Remove a package
pnpm remove lodash

# Remove a dev dependency
pnpm remove -D @types/lodash

# Remove a global package
pnpm remove -g serve

Update packages

# Update all (respecting semver ranges in package.json)
pnpm update

# Update a specific package
pnpm update react

# Update to absolute latest (ignoring ranges)
pnpm update --latest

# Interactive update (pick which packages to update)
pnpm update --interactive

# Check what's outdated without changing anything
pnpm outdated
03

Run Scripts & Binaries

Run package.json scripts

Unlike npm, pnpm doesn't need run for custom scripts.

# Run a script (no "run" keyword needed)
pnpm build
pnpm test
pnpm dev
pnpm lint

# Explicit "run" also works
pnpm run build

# Pass arguments to a script
pnpm test -- --watch --coverage

# Run pre/post scripts
pnpm run prebuild

Execute binaries

dlx = download + execute (like npx). exec = run already-installed binary.

# Download and execute (like npx)
pnpm dlx create-next-app my-app
pnpm dlx degit user/repo my-project
pnpm dlx serve -p 3000

# Run a binary already in node_modules/.bin
pnpm exec tsc --noEmit
pnpm exec vitest run
pnpm exec eslint .
04

Workspaces (Monorepo)

Set up a workspace

Create pnpm-workspace.yaml at the repo root.

# pnpm-workspace.yaml
packages:
  - 'apps/*'
  - 'packages/*'
  - 'tools/*'

Filter commands to specific packages

The --filter flag is pnpm's killer feature for monorepos.

# Run in a specific package
pnpm --filter @myorg/web dev
pnpm --filter @myorg/api build

# Add a dependency to a specific package
pnpm add react --filter @myorg/web

# Run in all packages matching a directory pattern
pnpm --filter "./packages/**" test

# Run in all packages that changed since last commit
pnpm --filter "...[HEAD~1]" build

# Run in a package and all its dependencies
pnpm --filter @myorg/web... build

# Run in all packages recursively
pnpm -r build
pnpm -r test

Link workspace packages

Use workspace:* to depend on local packages instead of registry versions.

# Add a local workspace package as a dependency
pnpm add @myorg/utils@workspace:* --filter @myorg/web

# In package.json, it looks like:
# "dependencies": { "@myorg/utils": "workspace:*" }

# Add a dependency to the workspace root
pnpm add -w typescript
pnpm add -wD prettier eslint
05

Inspect & Audit

Understand your dependency tree

# List all installed packages
pnpm list

# List only direct dependencies (no transitive)
pnpm list --depth 0

# Why is a specific package installed?
pnpm why react-dom
pnpm why lodash

# Security audit
pnpm audit

# Fix known vulnerabilities
pnpm audit --fix

# List licenses of all dependencies
pnpm licenses list
06

Store Management

Manage the global content-addressable store

pnpm stores every package version once globally and hard-links into projects.

# Show where the global store lives
pnpm store path
# → ~/.local/share/pnpm/store/v3

# Clean unreferenced packages from the store
pnpm store prune

# Verify store integrity
pnpm store status

# Install a global package
pnpm add -g typescript tsx serve

# List global packages
pnpm list -g

# Manage Node.js versions via pnpm
pnpm env use --global 22
pnpm env list
pnpm env remove --global 20
07

Patching & Overrides

Patch a dependency

Fix a bug in a dependency without forking. Creates a patch file tracked in git.

# Start patching (opens an editable copy)
pnpm patch express

# Make your changes, then commit the patch
pnpm patch-commit ./path-to-modified-package

# Patches are stored in your project and auto-applied on install
# package.json → "pnpm": { "patchedDependencies": { "express@4.18.2": "patches/express.patch" } }

Override transitive dependency versions

Force all instances of a package to a specific version, even deep in the tree.

// package.json
{
  "pnpm": {
    "overrides": {
      // Force all lodash to this version
      "lodash": "4.17.21",

      // Only override lodash when used by express
      "express>lodash": "4.17.21",

      // Use a local package instead of registry
      "some-pkg": "link:./local-fork"
    }
  }
}
08

CI & Configuration

CI install (frozen lockfile)

Use --frozen-lockfile in CI to fail if the lockfile is out of date.

# CI install — fails if lockfile doesn't match package.json
pnpm install --frozen-lockfile

# Import lockfile from npm or Yarn
pnpm import

# Generate lockfile without installing packages
pnpm install --lockfile-only

GitHub Actions setup

# .github/workflows/ci.yml
steps:
  - uses: actions/checkout@v4
  - uses: pnpm/action-setup@v4
  - uses: actions/setup-node@v4
    with:
      node-version: 22
      cache: 'pnpm'
  - run: pnpm install --frozen-lockfile
  - run: pnpm build
  - run: pnpm test

.npmrc configuration

Project-level pnpm settings. Place in the repo root.

# .npmrc

# Auto-install peer dependencies (default in pnpm 8+)
auto-install-peers=true

# Hoist specific packages that need flat node_modules
public-hoist-pattern[]=*eslint*
public-hoist-pattern[]=*prettier*

# Shamefully hoist everything (migration escape hatch)
shamefully-hoist=true

# Use a specific Node.js version
use-node-version=22.0.0

# Strict peer dependencies (fail on mismatch)
strict-peer-dependencies=true
